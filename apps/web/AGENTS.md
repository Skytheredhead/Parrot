# Prototype Instructions

Run the local server yourself and open the preview in the browser available to this environment. Do not give the user server-start instructions when you can run it.

Before making substantial visual changes, use the Product Design plugin's `get-context` skill when the visual source is unclear or no longer matches the current goal. When the user gives durable prototype-specific design feedback, preferences, or decisions, record them in `AGENTS.md`.

When implementing from a selected generated mock, treat that image as the source of truth for layout, component anatomy, density, spacing, color, typography, visible content, and hierarchy.

## Selected direction

- Use a Discord-familiar dark community shell as the visual anchor: slim workspace rail, grouped spaces/resources, dense central content, member presence, and compact interaction affordances.
- Preserve the product distinction from the research: spaces show substantial posts rather than a stream of every message; one post contains several bounded named threads; decisions/tasks retain source context; `Needs you` stays separate from unread activity; agents are labeled, quiet, scoped, and inspectable.
- Do not copy Discord branding, proprietary assets, exact icons, or distinctive trade dress. Do not collapse the product back into channel-first chat.
- Selected visual target: `reference-community-post-feed.png`.
